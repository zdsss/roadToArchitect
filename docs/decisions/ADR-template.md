## 示例：ADR-001-storage-data-structure.md

# ADR-001: 使用 Python dict 存储商品数据

**状态：** 已批准

**日期：** 2026-03-26

**背景**

Sprint 1 需要在内存中存储商品数据，并支持按 ID 查询。
当前商品数量预计不超过 10,000 个。

**决策**

使用 Python `dict`（key 为商品 ID）作为内存存储结构。

**考虑过的方案**

| 方案 | 优点 | 缺点 |
|------|------|------|
| dict（选中） | O(1) 按 ID 查找 | 按名称搜索仍是 O(n) |
| list | 顺序遍历直观 | 按 ID 查找是 O(n) |

**理由**

1. 按 ID 查询是最频繁的操作，dict 提供 O(1) 查找
2. list 的 O(n) 查找在商品数量增长时会成为瓶颈
3. dict 的内存占用与 list 相近

**后果**

正面：
- 按 ID 查询是常数时间，不随商品数量增长

负面：
- 按名称搜索仍是 O(n)，数据量大时需要专门的搜索索引

**迁移信号**

- 当商品数量超过 100 万，且按名称搜索成为性能瓶颈时，考虑接入 Elasticsearch
